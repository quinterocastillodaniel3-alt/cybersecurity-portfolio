## 📌 Project Description
In this project, I acted as a security analyst responsible for writing code to automate the analysis of login attempts made to a specific device. The primary objective was to create and assign variables to keep track of security-related information, such as device IDs, approved usernames, and login statuses.

---

## 🔍 Phase 1: Variable Assignment & Data Types
During this lab, I practiced defining multiple variables and verifying their underlying data types.

*   **String Variables:** Assigned the value `"72e08x0"` to a `device_id` variable and verified its type as `<class 'str'>`.
*   **List Variables:** Created a `username_list` containing approved users (e.g., `"madebowa"`, `"jnguyen"`) and checked its type as `<class 'list'>`.
*   **Integer Variables:** Tracked login limits by assigning the value `3` to `max_logins` and `2` to `login_attempts`, confirming their types as `<class 'int'>`.
*   **Boolean Variables:** Assigned `False` to `login_status` to represent whether a user is logged in, verifying its type as `<class 'bool'>`.

## 📝 Phase 2: Key Takeaways
*   The `=` assignment operator is used to assign or reassign specific values to variables.
*   The `<=` comparison operator evaluates whether one variable is less than or equal to another.
*   The `type()` function determines the data type of the value stored within a variable.
*   The `print()` function displays information, including direct values, variables, or Boolean evaluations.
